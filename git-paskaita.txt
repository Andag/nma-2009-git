Teorija:

git – paskirstyta versijavimo sistema



Įprastos komandos:

1. git init (ir --bare)
2. cache samprata; git status; git diff
3. git add; git commit
4. git log
5. failų atkeitimas (revert)


Šakos (branches) ir žymos (tags)
1. gitk
2. šakų kūrimas, trynimas
3. šakų jungimas (merge)
4. rebase
5. žymėjimas (tags)


Kooperavimasis:
1. Tiesioginis pullinimas iš kaimynų.
2. Tiesioginis pushinimas pas kaimynus.
3. Pakeitimų siuntimas el. paštu.
4. Atsiųstų paštu pakeitimų pritaikymas.


Egzotika:
1. git commit --interactive
2. git commit --amend
3. landžiojimas po vidines struktūras
4. git grep
5. git bisect
