Teorija:

git – paskirstyta versijavimo sistema


Įprastos komandos:

1. git init (ir --bare)
2. index samprata; git status; git diff
3. git add; git commit
4. git log
5. git blame
5. failų atkeitimas (revert)
6. git reset
7. .gitignore


Šakos (branches) ir žymos (tags)
1. gitk
2. šakų kūrimas, trynimas
3. šakų jungimas (merge)
4. rebase
5. žymėjimas (tags)
6. grįžimas į praeitį (checkout)


Kooperavimasis:
1. Tiesioginis pullinimas iš kitų aplankų / kaimynų.
2. Tiesioginis pushinimas į kitus aplankus / pas kaimynus.
3. Pakeitimų siuntimas el. paštu.
4. Atsiųstų paštu pakeitimų pritaikymas.
5. Konfliktų sprendimas.
6. „remote“ sąrašas


Egzotika:
1. git commit --interactive
2. git commit --amend
3. git stash
4. git grep
5. git bisect
6. landžiojimas po vidines struktūras
